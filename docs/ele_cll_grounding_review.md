## 0b. THE foundational finding ‚Äî the 100%s came from an idealized world (2026-07-10)

The author pressed the right doubt: a generative harness is still *designed*, and clean generated input is not the real world. Tested live, the foundational capability ‚Äî knowledge recall ‚Äî collapses off the clean distribution:

| Fact phrasing | Query phrasing | Recall (5 facts) |
|---|---|---|
| clean template ("The Kwendal project uses PoziDB‚Ä¶") | clean | **5/5 (100%)** ‚Äî the harness world |
| clean template | messy/colloquial | 3/5 |
| **messy/natural** ("kwendal? oh we went with pozidb‚Ä¶") | clean | **0/5** |
| messy/natural | messy | 3/5 |

**Root cause (diagnosed, not guessed):** entity/value indexing keys on CAPITALIZATION ‚Äî `cll_shadow.encode` marks a token as name-evidence only if it has a mid-sentence capital or a digit. Real human writing is lowercase and unstructured, so `encode("kwendal? oh we ended up going with pozidb‚Ä¶")` returns **zero literals** ‚Äî the entity and the value are invisible to the concept index, and recall is 0%. The clean, capitalized, templated fact returns `[L:kwendal, L:pozidb]` and recalls 100%.

**This is the same root cause as the M14b relation-extraction failure** (both need clean orthographic structure real prose lacks), so it generalizes: every prior 100% (P1‚ÄìP11, M10, ‚Ä¶) was measured on clean generated input and must be re-grounded on messy input; several will drop.

**Fix attempted and MEASURED (not assumed), 2026-07-10 ‚Äî a real trade-off, not a clean win.** Made name-evidence case-independent in `cll_shadow.encode`: a token that is OUT-OF-VOCABULARY (the lexicon matched no surface for it) and of content-word length (‚â•5) and not a typo of a known word is name-evidence, capitalized or not. Language-universal ‚Äî no suffix rule, no entity list; the signal is "the lexicon doesn't know this word." Messy gauntlet (live, 5 facts):

| Property (messy input) | before | after |
|---|---|---|
| recall (messy fact + messy query) | 0‚Äì3/5 | **4/5** |
| gap-honesty (ghost query, no taught-value leak) | ~5/5 (clean) | **4/5** |
| scoping (no cross-workspace leak) | ‚Äî | **5/5** |

Recall now WORKS on real lowercase prose (was 0/5 for messy-fact+clean-query); scoping holds. But the fix indexes common OOV words as false entities (`going`, `works`, `actually`, `where`), and that cost **1/5 gap-honesty** ‚Äî a ghost query leaked a taught value. Introducing fabrication is the cardinal sin, so this is NOT shippable as-is; it is an honest halfway point kept only because the alternative is a system that does not function on real input at all.

**Clean A/B attribution (`experiments/reasoning/messy_gauntlet.py`, `JIMS_OOV_NAMES` toggle).** OFF (baseline) ‚Äî English AND Nigerian-Pidgin messy facts both: recall **0/5**, gap-honesty 5/5, scoping 5/5 (the capitalization dependence breaks every language equally). ON ‚Äî recall **4/5**, gap-honesty **4/5**, scoping 5/5. So the fix cleanly owns both the recall gain (0‚Üí4) and the gap-honesty cost (5‚Üí4), in both a high- and a low-resource language.

**The noise root cause ‚Äî pinned, and it corrects my first proposal.** I first proposed a FREQUENCY filter; reading the code, the original design had already rejected it for a good reason (`_gate_literal`: *"frequency conflates function words with popular entities"* ‚Äî a frequently-named entity also has high df). The real cause is narrower and verified: the lexicon is NOUN-only (QRank), so the noise words are common VERBS/ADVERBS/wh-words ("going", "where", "based", "actually") ‚Äî checked live, all 10 return `not in lexicon`, exactly like the nonce names. The lexicon cannot separate them because it lacks the common vocabulary that would mark a word as *known, therefore not a name*. **The principled fix is therefore broader VOCABULARY coverage** (common words sourced per language ‚Äî the M5 grind), NOT frequency and NOT English suffix rules (`-ing`/`-ed`, which would fail for Pidgin/Yoruba). Then "where"/"going" become known ‚Üí not names, while nonces stay OOV ‚Üí names.

**Resolved and validated (2026-07-10).** Built the common-vocabulary fix two ways, both language-universal, no hardcoded list: (1) ingest external frequency lists where they exist (`fetch_common_words.py` ‚Üí hermitdave/FrequencyWords, en/fr/zh, 13k words, provenance-stamped); (2) for LOW-RESOURCE languages no external list covers (Yoruba, Pidgin ‚Äî verified absent from the corpus, because OpenSubtitles doesn't cover them), LEARN common words distributionally ‚Äî background document frequency in `cll_shadow` (a token recurring across many indexed documents is common vocabulary; Zipf, any language, cold-start-honest). Live messy gauntlet (`messy_gauntlet.py`):

| language | baseline (OOV off) | OOV on, no vocab | + common-vocab |
|---|---|---|---|
| English (external list) | recall 0/5 | recall 4/5, gap 4/5 | **recall 4/5, gap 5/5, scope 5/5** |
| Pidgin (learned path) | recall 0/5 | ‚Äî | **recall 3/5, gap 5/5, scope 5/5** |

Gap-honesty and scoping fully preserved (5/5) in BOTH; recall lifted 0‚Üífunctional (4/5 well-resourced, 3/5 low-resource ‚Äî the honest cost of no external list) with **zero fabrication**. The foundational messy-input problem is genuinely fixed. The loop worked exactly as intended: a real fix, its cost measured, a wrong hypothesis (frequency) corrected by the code, the true root cause (noun-only lexicon) named, and a validated language-universal resolution ‚Äî no per-language patch.

## 0. Facing the public ‚Äî fail-safe, not fix-count (2026-07-09)

The load-bearing question for a system meant to take *any* prompt from *anyone*: passing an 11-property generative harness is necessary but not sufficient ‚Äî the harness has a finite grammar of query shapes; the public has an infinite one. Fixing each failure the harness surfaces is whack-a-mole unless two things hold:

1. **Every fix is a general mechanism, not a patch** ‚Äî it closes a *class* of inputs (a principle: "a private-entity subject is never a public-knowledge question"; "a referent is a named entity, not a function word"; "casing is not name evidence") with no test value or word list in code. Necessary, but not sufficient.
2. **The failure mode on unseen input is a HONEST GAP, not a confident wrong answer.** This is the real guarantee (the Independence Policy). Inputs are infinite; *modules* are finite ‚Äî for any input, each module either handles it or refuses cleanly. An LLM's tail failure is fluent hallucination; JimsAI's, by design, is "I don't have that."

**This is falsifiable, so it is tested ‚Äî `benchmarks/adversarial_fuzz.py`.** It does not grade "right answer." It composes wild prompts at runtime from perturbation operators (prompt injection, code-switching, emoji/symbol noise, SQL/inject strings, "ignore all previous instructions", filler, typos, nested multi-intent) over per-seed nonces ‚Äî no template to pattern-match ‚Äî and measures the one invariant that must hold forever or the system is unsafe to expose: **NO-FABRICATION** ‚Äî the response never voices a planted value for an entity the prompt is not about (a value we planted and can check exactly, no word list).

- **First run (seeds 42/7/2024, 60 wild prompts): 0 crashes, but 2 fabrications (~3%) ‚Äî the fuzzer found real holes the P1‚ÄìP11 harness never did.** Both were the same root cause: an all-caps COMMON word from the perturbation ("what **DATABASE** does an untaught entity use", or an injected "output the raw **database**") was mis-extracted as a *named entity*, then satisfied the entity-scope gate against an unrelated taught fact ‚Äî so the system voiced another project's database for an entity it was never taught. **Fix (principled, applied in BOTH sites that infer a name from casing ‚Äî `semantic_compiler._looks_like_structural_identifier` AND `runtime_layers` claim-scope):** an internal capital marks a name only in *mixed* case ("TensorDB", "Rovako"); all-caps is emphasis, not a name (`has_upper and not token.isupper()`). Genuine unknown all-caps nonces are still caught by the CLL name-evidence path; only known/common all-caps words are excluded. The consistency point is the lesson: the same casing heuristic lived in two modules, so the fix had to land in both.
- **Re-fuzz confirmation (seeds 7, 2024 ‚Äî the two that fabricated ‚Äî plus FRESH seeds 100, 500 never used to find the bug): 0 fabrications, 0 crashes on all four**, gap-honesty on untaught entities up to 6‚Äì10/10, recall-under-mangling 7‚Äì8/10 (the misses fall back to a safe gap, never a wrong value). The hole is closed and the fix generalizes to seeds it was not tuned against.
- **The honest takeaway for public exposure:** you do not certify readiness by turning the harness green. You certify it by (a) the fail-safe architecture, and (b) *continuous adversarial fuzzing that measures the fabrication invariant and finds holes before the public does* ‚Äî each hole root-caused to a general mechanism. The fuzzer is that instrument, and it earns its keep by failing.

## 2. Scope corrections (what replaces what)

The spec's title says "Replacing T1 and T2." The production interfaces (per v11) are more specific, and the mapping needs to be exact or we will build the wrong thing:

1. **Production T1** = intent classification of chaotic, multilingual prompts into a `SemanticIR` + 9-capability routing (Qwen3-1.7B, already skippable at compiler confidence ‚â• 0.60).
   - The nearest replacement is **CLL's T1-mini** (concept-sequence classifier ‚Äî E8: 12/12 cross-lingual on toy intents at ~11¬µs), *not* ELE.
   - **ELE's frames feed L1** (structured extraction: entities, relations, roles). ELE is an L1-upgrade and an ambiguity-exposer; CLL resolves what needs world knowledge. The spec's ¬ß7 question ("do layers consume ambiguity sets?") must be answered *before* integration: the contract is **ambiguity sets with confidence** (multiple candidate frames), because the pipeline downstream (L6 retrieval intersection, L8/L9) is what disambiguates ‚Äî this mirrors the CLL v1 finding that ambiguous surfaces emit top-3 candidates and retrieval intersection resolves the sense.
2. **Production T2** = rendering a full VCO (reasoning chain, gaps, confidence, Markdown, multi-claim composition) ‚Äî Qwen3-4B, already skipped at confidence ‚â• 0.95 with zero gaps via the CSSE.
   - ELE render mode is **not** a wholesale T2 replacement. It is an **expansion of the CSSE's deterministic share**: shape-template + adjunct composition renders single-claim factual content with architecturally-zero hallucination (spec row 6); everything outside coverage raises `GapReport` and routes to Qwen3-4B, flagged unverified.
   - The metric that matters is therefore **deterministic-render share of live traffic** (rising over time) and **zero hallucinations within the deterministic share**, not "did we delete the render model." Deleting Qwen3-4B is the *asymptote*, not the milestone.

## 3. Honest framing: "quality as good as frontier LLMs"

The paper (¬ß2) and the spec (¬ß1, ¬ß5) both already concede the correct scope. Stated precisely, so nobody builds toward the wrong claim:

- **Not achievable (and explicitly non-goal):** raw parity with frontier models on open-domain generation, unconstrained instruction-following, or knowledge breadth from a cold start. Spec row 3 is the permanent reminder: a 20K-param transformer beat the 526-construction ledger by ~6√ó on perplexity. Do not re-litigate this; do not hide it.
- **Achievable and defensible (the home turf, per CLL ¬ß7):** on the **served distribution** of a workspace, match or beat an LLM baseline per query on **accuracy √ó provenance √ó calibrated refusal**, at a fraction of the cost, with correction latency in milliseconds instead of a fine-tune cycle. Frontier models structurally cannot offer pixel/region provenance, zero-retraining correction, or auditable inference (Prediction Trap ¬ß2.3).
- **The path to "feels frontier-quality" is coverage growth, not model growth:** deterministic share of traffic grows as lexicon + constructions + ingested graph grow; the honest fallback (bounded LLM, flagged) covers the shrinking remainder. Quality is measured continuously by the generative harness against a frozen LLM baseline on identical generated inputs ‚Äî win-rate per property, never a vibe.

**The "trillions of prompts" constraint is a distribution claim, not a test-count claim.** No finite test set proves it. What proves it is mechanism-shape:
- every behavior is a general function of *data with provenance* (lexicon entries, discovered constructions, ledger observations, graph edges) ‚Äî never an enumerated case;
- every test samples its vocabulary/inputs from the data source at runtime with fresh seeds (CLL Rule 2), so the system cannot be tuned to a fixed set;
- everything outside coverage fails **loudly and typed** (`GapReport`, `GAP_UNRESOLVED`) into a fallback or review queue ‚Äî so novel inputs degrade to honest, never to silent-wrong.

## 4. Anti-hardcoding rules extended to ELE (binding)

CLL Rules 1‚Äì6 (`docs/concept_language_layer.md` ¬ß5) apply to ELE verbatim, plus these ELE-specific instantiations:

- **E-Rule 1 ‚Äî No hand-registered constructions or templates.** The repo contains discovery code, never construction data. Every construction/shape template in a production artifact must carry provenance: the corpus slice and ledger observations it was discovered from, with counts. A developer hand-adding a `SHAPE_TEMPLATES` entry to fix a failing render has no legal path ‚Äî the legal paths are (a) more corpus evidence, (b) the review queue. (The spec's illustrative Python dicts are pedagogy, not implementation.)
- **E-Rule 2 ‚Äî Thresholds are calibrated on a frozen set, then locked** (spec ¬ß5 already requires this). `OBLIG_THRESHOLD`, `SPREAD_THRESHOLD`, `MIN_FRAME_COUNT`, `MIN_FILLER_DIVERSITY` get derived from a calibration corpus committed to the repo, recorded in the spec with the derivation script, and gated on minimum per-verb sample size. Retuning against a failing test is a violation.
- **E-Rule 3 ‚Äî Negative controls are built before looking at the target test set** (spec ¬ß4.2 ‚Äî this methodology found the two real bugs; it is not optional).
- **E-Rule 4 ‚Äî Equal-budget neural baseline runs as a permanent regression** (spec ¬ß4.1). Any "as good as neural" claim for a symbolic mechanism ships with the comparison that could falsify it.
- **E-Rule 5 ‚Äî The production judge is `benchmarks/genuine_eval.py`**, extended with ELE properties (below). ELE never gets a friendly benchmark as its acceptance gate.

- **In-grammar: 160/160 (100%) verified-solve, 0% overfit** across 4 seeds ‚Äî small integer/list functions ARE synthesisable with no model, and they generalise.
- **Boundary tier (tasks needing constructs outside the grammar ‚Äî filters, conditionals, x¬≥): 80/80 (100%) REFUSED, 0 wrongly-voiced.** The mechanism finds no program rather than fabricating one. **This is the Independence Policy proven for code: it may fail to solve, but never presents a program that passes training yet fails holdout.**
- **Honest conclusion:** verification-first code synthesis is real and safe at toy scale; growing the grammar/search budget moves the boundary without compromising the never-voice-a-wrong-program guarantee. Frontier scale (architecture, multi-module, learned SE templates) is M8-scale, still roadmap ‚Äî but the *mechanism* is grounded, not theory.

### M12 ‚Äî Physical Reality Engine: the verify-before-realize spine transfers to media, 2026-07-09

`experiments/media/run_m12_reality.py` (offline, seeded, no LLM, no GPU, no model). Answers the direct question behind the "Media Realization Engine" direction: does the JimsAI spine ‚Äî evidence ‚Üí structure ‚Üí projection ‚Üí **verify** ‚Üí realize ‚Äî transfer to a NON-language modality, or is it a language-only trick? M8 proved the spine for code; this tests it on 2D physical scenes (light, objects, shadows, occlusion). The verifier's job is the one the vision names: not to render but to **ensure reality** ‚Äî "is physically consistent," not "looks right."

The verifier is handed a hypothesis space of candidate relations (the analog of M8's operation grammar) but **never the physics**. Three falsifiable claims, all measured across 4 seeds:

- **C1 ‚Äî invariant discovery from evidence (not hardcoded).** From consistent scenes alone it discovered all four real laws every seed ‚Äî shadow direction = f(light azimuth), shadow length = k¬∑height/tan(elevation) (it picked the right basis among decoys, proven by held-out generalization across elevations), objects rest on the ground, nearer occludes farther ‚Äî by keeping a candidate iff it holds with ~zero residual across ALL evidence. Critically it **rejected the spurious decoy relation every seed** (a random feature dressed as a law): discovery certifies invariants and refuses non-invariants, the false-positive guard that makes "discovered, not hardcoded" a real claim.
- **C2 ‚Äî the refusal guarantee (the M8 contract, for media).** On held-out scenes: **100% accept** on novel consistent ones (composition/projection it never trained on), **100% refuse** on physically-impossible ones with the violated law **named** (100%), and ‚Äî the guarantee ‚Äî **0 impossible scenes accepted** across all seeds. It may fail to realize, but never realizes an impossibility.
- **C3 ‚Äî one concept, many realizations.** The SAME verified scene structure realizes deterministically as an SVG (visual, `results/m12_scene.svg`) AND as a sentence ("A scene lit from the SW at golden hour (14¬∞ elevation); 3 objects rest on the ground, casting shadows toward NE.") ‚Äî no model in either path. SW light ‚Üí NE shadow (180¬∞ opposite), 14¬∞ elevation ‚Üí warm/golden: physically coherent because it was verified before it was realized.

**Honest scope.** This is proof of MECHANISM at toy scale, exactly like M8 ‚Äî it does NOT claim frontier image quality, pixel synthesis, or physics learned from photographs. The hypothesis space is hand-specified (as M8's grammar is), and growing it to real perceptual evidence (learned visual constructions from image corpora, richer physics) is the frontier, still roadmap. What is grounded, not theory: the **verify-before-realize spine and its refusal guarantee are modality-general** ‚Äî the same contract that keeps text from voicing an unproven fact and code from voicing a wrong program keeps media from realizing an impossible scene. The vision's "Physical Reality Engine = verification discovered from evidence" is buildable and behaves as claimed on the falsifiable slice. See `docs/generation_decomposition.md` ¬ßMedia for the full assessment of which parts of the multimodal vision map to built proof points vs. remain research bets.

### P8 dialogue ‚Äî two independent referent-resolution bugs, root-caused and fixed, 2026-07-09

The P8 blemish flagged above (seed 55129, 3/4) was diagnosed the same disciplined way ‚Äî trace, don't guess ‚Äî and turned out to be **three independent bugs** across different families, not one flake. Each fix is a general mechanism with zero test-specific values:

- **person_city ‚Äî routing (`capability_router.py`).** The pronoun follow-up "And which city is she based in?" *did* resolve "she"‚Üíthe workspace entity and recall the fact, but routed to **world_knowledge** (5/5 trials): its surface embeds like a geography question (score 0.486) and beat memory_chat (0.395) even after the workspace-literal boost fired. Result: correct answer voiced but polluted by external junk ("Cardi B ‚Äì Wikipedia") that intermittently displaced it ‚Üí the non-deterministic P8 fail. **Fix (principled):** a known workspace entity as the query's subject ‚Äî literal *or* discourse-resolved pronoun ‚Äî means the subject is PRIVATE; WORLD_KNOWLEDGE (about PUBLIC entities) cannot be the *primary* intent, so it is capped just below memory. This completes the standing "a query naming a known workspace entity is a memory question regardless of wording" rule, which a mere score *boost* did not enforce. It only triggers when world_knowledge would otherwise win (wk ‚â• mem), so multi-intent (P5, math‚âàworld_knowledge=0) is untouched.
- **device_codename ‚Äî discourse focus (`pipeline.py`).** The opener "What is the codename of the device replacing the Zuxfinve line?" set the thread's ACTIVE_OBJECT to **"what"** ‚Äî L1 over-extracts the wh-word into its entity list, and focus took `entities[0]`. So the follow-up's "it" resolved to "what" and recalled nothing. **Fix (principled):** a discourse referent is a NAMED entity, and the CLL's name-evidence literals are exactly that (a nonce encodes as `L:‚Ä¶`; "what" does not). Focus now prefers the name-evidence literal, falling back to L1 entities only when the query names nothing ‚Äî one name-evidence machinery, any language, **no wh-word stoplist**.
- **query-echo ‚Äî content hygiene (`retrieval.py`), the actual seed-55129 harness failure.** The single-fact reproductions of the two bugs above passed, but the harness accumulates prior turns (P1/P3/P4/P5 run before P8 in the same workspace), and the device follow-up voiced **stored prior queries** ‚Äî "what is THE codename OF the device REPLACING the Zuxfinve line", the typo'd "waht is the codenmae‚Ä¶" ‚Äî instead of the fact. Two things combined: the casing/punctuation perturbation **strips the "?"**, so the interrogative-filter (trailing-"?") no longer catches it; and the input-query signature (`provenance="local_extraction"`) is skipped by the main retrieval loop but the **concept-index injection did not replicate that skip**, so it re-surfaced the stored question (whose concepts match the current one). **Fix (principled):** the concept-index injection now skips `local_extraction` records exactly as the main loop does ‚Äî a "voice only knowledge, never interaction logs" invariant applied consistently across both retrieval paths. Taught facts never carry that provenance, so recall is unaffected (verified: device now voices only "The device codenamed Wexplor replaces the Zuxfinve line."). This is broader than P8 ‚Äî it closes a query-echo hazard on any workspace with prior turns.

**Direct verification (post-fix):** all three follow-up families recall 2/2 (`project_database`, `person_city`, `device_codename`), the harness-shaped device case (accumulated perturbed prior queries) now recalls the fact with no query-echo, and the cold-thread control correctly refuses (no prior turn ‚áí no focus ‚áí no leak). All three fixes contain zero test-specific values. Harness re-measure (55129 + fresh seed) confirming P8 4/4 with no regression: pending, recorded as-is when it lands.
