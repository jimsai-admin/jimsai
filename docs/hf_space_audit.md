## _wake_hf_space() Call Sites

| # | File Path | Line | Type | Notes |
|---|-----------|------|------|-------|
| 1 | `services/api-gateway/app/main.py` | 20 | **definition** | `async def _wake_hf_space() -> None` — fire-and-forget background task that pings `JIMS_EMBEDDING_SERVICE_URL/health` up to 15 times (≈4 min) to pre-warm the HF Space embedding service before the first real request |
| 2 | `services/api-gateway/app/main.py` | 45 | **call / lifespan hook** | `asyncio.create_task(_wake_hf_space())` inside the `@asynccontextmanager async def lifespan(app: FastAPI)` — called at FastAPI startup via the `lifespan=lifespan` parameter on the `FastAPI(...)` constructor (line 49) |

### Context: lifespan hook (lines 43–48)

```python
@asynccontextmanager
async def lifespan(app: FastAPI):
    # Kick off HF Space warm-up in the background — don't block startup
    asyncio.create_task(_wake_hf_space())
    yield
```

The `FastAPI` app is then constructed with:

```python
app = FastAPI(title="JIMS-AI api-gateway", version="0.1.0", lifespan=lifespan)
```

### No other call sites found

A full codebase search (excluding `.git/`, `node_modules/`, `__pycache__/`, `.kiro/`)
found **no other files** that define, call, or import `_wake_hf_space`.

Specifically checked:
- No `import` of `_wake_hf_space` anywhere
- No additional `_wake_hf_space()` calls outside `main.py`
- No `@app.on_event("startup")` handlers referencing `_wake_hf_space`
- `services/api-gateway/app/lambda_handler.py` uses `Mangum(app, lifespan="off")` — suppresses the lifespan hook in Lambda deployments, meaning `_wake_hf_space` is **not** invoked when running under AWS Lambda

### Action required (task 10.5)

Per requirement 9.9, both entries above are **marked for removal** once the Modal backend is in place:

1. Delete lines 20–38 (the `_wake_hf_space` function body) from `services/api-gateway/app/main.py`
2. Delete line 45 (`asyncio.create_task(_wake_hf_space())`) from the `lifespan` context manager
3. If the lifespan context manager has no other startup work, the entire `lifespan` function and the `lifespan=lifespan` kwarg on the `FastAPI` constructor can also be removed
4. No Modal-equivalent startup ping is needed — Modal containers are kept warm via `min_containers=1`
